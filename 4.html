Week 4 Project: Prosthetic Hand

October 13th and 17th

For the Week 4 Project, I was introduced to a 3D-printed prosthetic hand. We have 3D printed the components of the open-source prosthetic hand and assembled them using a kit and some documentation manuals.

1. Online Community

e-NABLE Hub is an online community of 3D-printed hands and arms for prosthesis devices. The website has a forum to help each other, a chapter to collaborate with nearby volunteers, and resources to aid volunteers in creating the perfect device at every step of their journey.

In a similar manner, there is NIH's (National Institute of Allergy and Infectious Diseases) 3D e-NABLE collection. This website contains a lot of assistive technology designs that are freely available. The collection has many 3D files of prosthesis devices for different needs based on one's physical characteristics.

For this project, e-NABLE Phoenix Hand v3 will be used. This latest version was created by remixing two previously famous devices: Phoenix Hand v2 and Unlimited Phoenix Hand. This device is available from both websites: e-NABLE Hub and NIH. Both websites contain almost identical information. They provide links to the 3D files, instruction manual, video tutorials, and assembly kit for purchase. However, the first website contains a little more information and an updated 3D file. Firstly, the e-NABLE Hub provides information about the device's maturity, cost, popularity, difficulty, and grip strength. For all five categories, e-NABLE Phoenix Hand v3 received the best rating, signaling that this prosthesis device is highly mature, costs little, is popular, is easy to assemble, and has a strong grip. Secondly, e-NABLE Hub has updated the 3D file to make the assembly process a bit easier. The six different pins required for the assembly can be hard to differentiate due to their similarity. Hence, e-NABLE Hub has added a label from A to F so that the volunteer would not accidentally use the wrong pin. Overall, both websites provide similar information and resources for e-NABLE Phoenix Hand v3, but e-NABLE Hub goes further and provides a polished version of the device.

I have also explored other devices on their websites and other 3D repository site. On NIH collection, some of the unique prosthesis devices include bike handle adapter and viola bow holder . On a 3D repository site Thingiverse, some of the unique prosthesis devices include an adapter to hold the tool, a remixed version of e-NABLE Phoenix Hand v2 as a Superman theme, and an adapter to hold a pencil. Clearly, there are many open-source prosthesis devices for different needs available online.

2. Video Tutorial vs Written Instruction

For this project, my partner and I decided to assemble the widely respected e-NABLE Phoenix Hand v3. To help us assemble, we reviewed two assembly manuals: a YouTube video tutorial from 3D Universe and a written instruction from Enabling the Future. The video tutorial walks through each step of the journey. In this 36-minute long video, the host assembles the device from the start to the end while describing her action and providing some tips and warnings. The written instruction provides step-by-step instructions on the assembly process for the second version of the prosthesis hand. In this 44-page manual, there are paragraphs of instruction on each page with some 3D visual rendering of each process. Overall, both manuals are detailed and provide good information, but there are some differences between them, and there are some areas of improvement.

Foremost, both manuals are detailed and helpful. They both include visual elements and some sort of language. The video shows the host going through the steps, while the written instruction includes a 3D visual rendering of each process. And, in the video, the host talks through the step, while in the written instruction, the steps are written down in paragraphs.

However, there are differences in how each manual approaches the assembly process and the content of each manual.

Firstly, in the written instruction, different color was used for each of the different components so that each component could be distinguished from the other. However, in the video, all the components of the 3D printed parts were in one color: brown. Hence, I found it difficult to distinguish each of the components in the video, so if I could change the video, I would print each of the components in a different color, similar to how the written manual has approached this. Therefore, the video can improve the visual aspect of their delivery.

Secondly, in the video, the host frequently tested out the device to ensure that the build was coming in good shape. For example, the host tried out both dental bands and examined which band fit better before making a commitment. However, in the written instruction, there was no mention of taking a pause and checking if the assembly was taking place well. The written instruction could have done a better job documenting a build step by asking the volunteers to test out if the build was assembled correctly. The instruction should explain how the hand should look like and what it should be able to do and should not be able to do if the assembly is done correctly. For instance, similar to the step taken in the video, the written manual should ask the volunteers to try out both dental bands and explain that they should pick a dental band that allows the finger to be folded completely without having to apply too much force. Hence, the written instruction can be improved by being more descriptive about the device rather than merely going through all the steps.

The other few differences between the video tutorial and the written instruction are minor. For instance, in the written instruction, the index and middle fingers were attached to the hand before attaching the ring and pinky fingers, and the instruction contained a link to purchase the assembly kit. These minor differences are not significant and did not impact the build process.

One of the most confusing aspects of those instructions is the mechanics of the strings. Initially, I was not sure how the strings all worked together. When I was watching the video and reading the instructions, I was confused as to why there are only three strings and not five strings. And, there are lots of holes that the string need to go through, and it was confusing to keep track of all of the holes. These confusions can be addressed by giving the audience an overview of the mechanics of the strings and a detailed diagram of where all the holes the three strings need to go through.

Taking a step back, the video tutorials and the written instructions are good, but I think I could create a better video tutorial and better written instruction. Besides the points mentioned above, they can be better by being more concise. I think 36 minutes is too long to watch, and 44 pages are too long to read. The video should be under 10 minutes, and the written instructions should contain less than 500 words. For the video, I would use animation to better visualize the moving components and use visual elements, such as arrows and circles, to highlight some important areas. I would make the video scripted so that it is easier to understand. For the written instruction, I would put as few words as possible, like a wordless Lego instruction, by using visual elements, such as arrows and before and after pictures. I would only write long sentences when needed, such as writing a warning to put a pin through the index finger and middle finger at the same time. By taking these steps, the video tutorial and the written instruction can be more accessible to a broader audience, such as to the younger audience and non-English-speaking audience. Lastly, the video tutorial and the written instructions did not give a good overview of this build, and there was a lack of context. I was a bit confused about this build, because they did not give a good context around it, and I was not sure what the problem it was trying to solve and how it was trying to address it, so I would include them in my version.

3. Printing the Components

For this project, my partner and I chose e-NABLE Phoenix Hand v3 as the prosthesis device to assemble. The 3D file (STL) was downloaded from the NIH website for free, and it took multiple prints to successfully print all the parts. Initially, I tried to use a Raise E2 3D printer. I modified the scale of the components by 150%, because the professors recommended us to print a bigger hand as it is easier to assemble. I was initially confused as to why this was the case, but when I think about it, this is similar to how big Lego bricks for toddlers are easier to assemble than normal-sized Lego bricks. After increasing the scale, I laid out the pieces on Raise's proprietary CAM software IdeaMaker. It took some time to lay all the pieces in the limited space, because there were a lot of components to print, the size of the printable area was not big, and I wanted to print all the components in one session. However, I soon found out that there was an issue with the nozzle, and without a mechanical fix, the printer would not function. So, I used Ender and Cura to print them. There were some hiccups along the way, such as the filament ran out midway, the print lifted away from the bed, some layers did not stick well, and the scale was not increased. Gladly, most of the issues were fixed by reprinting them.

4. Bend the Plastic

One of the 3D-printed parts is used to hold the client's arm, but it has to be heated and deformed to a curve. Due to safety concerns, our TA heated and bent the part for everyone. It was very cool to see the plastic bend without melting.

5. Assembling the Hand

Due to the scheduling and timing issues, my partner and I used a pre-printed piece to assemble the hand. Along with the 3D printed parts, we used an assembly kit purchased for this prosthetic device, and a scissor and a screwdriver were the necessary tools needed to assemble it. For the guidance, we mainly used the written instructions to assemble it, but since the instruction was written for the previous version, we relied on the video for some parts, such as attaching the string. The overall assembly was smooth, but there were some incidents. Firstly, we made a mistake when attaching the pin for the middle and index fingers. We attached the pin after lining up the index finger without also lining up the middle finger. This was a problem because the middle finger cannot be attached to the hand. To pull the pin out, we had to use some sharp objects to forcibly push it out. Secondly, we were not sure how to tie the string at both ends. We wanted it to be sturdy, yet we did not know the best way to tie a single string rope, as we mainly have experience tying two strings together. Regardless, we tried our best and made multiple nodes to ensure that it would withstand great tension. Thirdly, we were confused about how the strings work and their role. We knew that the strings were meant to squeeze the hand but were not sure how it exactly worked. So, we looked at the video carefully and our classmate's finished product for reference. This helped us get a better sense of the idea of how the string should be attached and incorporated into the hand. Lastly, we struggled the most with calibrating the string to optimize the grip. The grip was loss, because we did not follow the instructions carefully and tie the string when the hand is folded at 30°. So, we had to untie the string and adjust the three metal screws. Some strings were tied so tightly that we cut them off and attached a new string. Overall, although it took more than two hours to assemble it, it was fairly smooth as any obstacles were overcome relatively quickly and simply.

6. Customized Hand for Sculpture

If I ever needed a prosthetic hand, I would want a customized version of a hand that is tailored to sculpturing, one of my favorite activities. For this art activity, several tools are needed: wire-end modeling, smooth modeling, carving, and detailing. These tools will be on each of the fingertips. Assuming that each hand can be controlled individually, such a prosthetic device can be beneficial when sculpturing because all the tools will be on their fingertip rather than having to drop off the tool, find the new one, and pick it up. This can save time. Also, having it on the fingertip could give greater control because it is built on the device and is sturdier and stable.
